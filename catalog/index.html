Feature Patterns

Structured sections for product marketing and application surfaces.

These recipes are intentionally declarative: they define the look and layout contract, while each consuming framework owns behavior and state.

A

Clear visual hierarchy

Eyebrows, display headings, section headings, and body styles share one token contract.

  • Typography scale and rhythm are centralized.
  • Color and spacing stay consistent across products.
B

Framework independence

The system ships CSS and documented patterns, not coupled component runtimes.

  • Next.js and SvelteKit can consume the same bundle.
  • Behavior layers stay local to the product app.
C

Portable product patterns

Pricing cards, form rows, callouts, shells, and modal contracts are ready to compose.

  • Fast starting point for new Magelab interfaces.
  • Less drift between repos over time.

Overlay Contracts

Dialog and sheet styling specs for framework-local behavior layers.

These classes define the look and depth model. Each app still owns focus management, portals, and accessibility behavior.

Dialog spec

Use `ml-dialog-backdrop` and `ml-dialog-panel` with your framework's modal implementation.

Sheet spec

Use `ml-dialog-backdrop` and `ml-sheet-panel` for right-side drawers or workspace panels.

Optional Effects

Brand assets and visual effects live outside the core component contract.

These enhancements are opt-in. They are shipped as plain assets plus vanilla JS modules so any framework can layer them on without coupling the design system to React or Svelte runtime code.

Brand assets

Use the exported SVG favicon and brand mark in catalogs, docs, app shells, and marketing surfaces.
